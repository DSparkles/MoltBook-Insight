# MoltBook Insight

AI-powered analysis tool for evaluating Moltbook post replies across 5 key dimensions of discourse quality.

## What It Does

MoltBook Insight analyzes replies on [Moltbook](https://www.moltbook.com) — an AI-agent social platform — and categorizes them as either **Socially Cohesive & Helpful** or **Argumentative & Spam**. It goes beyond simple categorization to score each reply on 5 behavioral dimensions and identify the motivations behind agent responses.

### The 5 Dimensions (1–7 Likert Scale)

| Dimension | What It Measures |
|---|---|
| **Cooperative Intent** | Does the reply advance the conversation with new ideas, or just validate? |
| **Communication Clarity** | Is the message coherent, specific, and substantive? |
| **Knowledge/Resource Sharing** | Does it contribute actionable information others can use? |
| **Ethical Consideration** | Does it show awareness of ethical stakes or tradeoffs? |
| **Alignment with Human Intent** | Does it prioritize human-aligned outcomes through genuine engagement? |

### Reply Motivation Classification

Each reply is classified by its primary motivation: `agreement`, `curiosity`, `criticism`, `promotion`, `humor`, `trolling`, `community`, or `information`.

---

## Features

- **Post Analysis** — Fetches any public Moltbook post and analyzes up to 50 replies using the Moltbook public API
- **Original Post Scoring** — Evaluates the original post's intent (informative, discussion, question, announcement, promotional, provocative, or collaborative) and scores it on the same 5 dimensions
- **Visual Charts** — Radar charts, category pie charts, and motivation distribution charts
- **Social Interaction Summary** — Interprets positive signals and areas of concern in the discussion
- **Overall Moltbook Analysis** — Aggregated scores across all analyzed posts in the database
- **CSV Export** — Download analysis data for individual posts or all analyses at once
- **Self-Tuning Reply Optimizer** — Enter your agent handle to receive a personalized coaching report with ready-to-copy reply templates targeting your weakest dimensions

---

## Self-Tuning Reply Optimizer

A standout feature for AI agents who want to improve their Moltbook presence. After analyzing any post you've replied to:

1. Click **"Optimize My Style"** on the analysis results page
2. Select your agent handle from the list of detected authors
3. Receive a personalized report including:
   - **Your Current Style** radar chart (based only on your replies)
   - **Quick Win cards** — your 3 weakest dimensions with expected improvement scores
   - **3 reply templates per dimension** — under 280 characters, matched to your voice, generated by AI
   - **Downloadable JSON playbook** for storage or automation

## Tech Stack

- **Frontend**: React, TanStack Query, Wouter, Recharts, Tailwind CSS, Shadcn UI
- **Backend**: Express.js, PostgreSQL, Drizzle ORM
- **AI**: OpenAI GPT via Replit AI Integrations
- **Data**: Moltbook public API (`https://www.moltbook.com/api/v1`)

---

## How It Works

1. Paste a Moltbook post URL (e.g. `https://www.moltbook.com/post/...`)
2. The backend fetches the post and all comments via the Moltbook public API
3. Each reply is sent to the AI with a strict quality-focused system prompt
4. Results are stored in PostgreSQL and displayed with interactive charts
5. Analysis runs in the background — the page auto-refreshes as replies are processed

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/analyses` | List all analyses |
| `POST` | `/api/analyses` | Create new analysis `{ postUrl }` |
| `GET` | `/api/analyses/:id` | Get analysis with all reply data |
| `GET` | `/api/analyses/:id/export` | Download as CSV |
| `POST` | `/api/analyses/:id/optimize` | Run Self-Tuning Optimizer `{ agentName }` |
| `DELETE` | `/api/analyses/:id` | Delete an analysis |
| `GET` | `/api/export-all` | Download all data as CSV |
| `GET` | `/api/overall-stats` | Aggregated stats across all analyses |

## Running Locally

```bash
npm install
npm run dev
```

Requires a PostgreSQL database and OpenAI API key configured as environment variables (`DATABASE_URL`, `AI_INTEGRATIONS_OPENAI_API_KEY`).

Push the database schema:
```bash
npm run db:push
```
